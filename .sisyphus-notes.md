# AI Tools Auto-Thread Implementation

## Summary
Implemented auto-thread creation for tool submissions (OCR, Searchable PDF, X-ray) on bare workspace routes with empty chat history, achieving parity with normal chat UX.

## Behavior Changes

| Scenario | Before | After |
|----------|--------|-------|
| **Tool submit on bare workspace + empty history** | Tool runs in threadless context (thread_id=null) | Auto-creates thread, runs tool in thread, navigates to thread |
| **Tool submit in existing thread** | Runs in thread (unchanged) | Runs in thread (unchanged) |
| **Tool submit on bare workspace + existing history** | Runs threadless (unchanged) | Runs threadless (unchanged) |
| **Tool request aborts (user navigates away)** | Optimistic entries removed, no navigation | Optimistic entries removed, no navigation (same) |
| **Tool request errors** | Error persisted in threadless context | Error persisted in thread, navigates to thread |

#### 1. Added mounted ref (line ~77)

Prevents setState-after-unmount warnings during navigation after tool completion.

#### 2. Updated cleanup effect (line ~133)
```javascript

  return () => {
    mountedRef.current = false;  // Mark unmounted
    toolAbortRef.current?.abort();
    toolAbortRef.current = null;
  };

```

#### 3. Refactored tool submit handler (line ~195-298)
- Extracted async logic into `submitTool()` function
- Added thread creation guard: `!activeThreadSlug && chatHistory.length === 0`
- Thread creation happens BEFORE optimistic entries (prevents race conditions)
- Passes `toolThreadSlug` to `AiTools.run()` (server already accepts threadSlug form field)
- Navigates to thread AFTER state updates complete (SUCCESS or ERROR, not ABORT)
- All setState calls guarded with `mountedRef.current` check
