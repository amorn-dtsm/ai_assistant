# Source Viewer Test Fixtures

This directory contains test fixtures for the source-viewer component. Each fixture is designed to test specific document type handling and text extraction capabilities.

## Fixture Overview

| Fixture | Type | Anchor Sentence | Purpose |
|---------|------|-----------------|---------|
| `digital-latin.pdf` | PDF (2 pages, text layer) | "The quick brown fox anchors here." | Test born-digital PDF text extraction |
| `digital-thai.pdf` | PDF (Thai text) | "การทดสอบระบบค้นหาข้อความภาษาไทย" | Test Thai text in PDF |
| `scanned.pdf` | PDF (2 pages, image-only) | N/A (no text layer) | Test OCR fallback for scanned documents |
| `simple.docx` | DOCX (h1, paragraphs, table) | "The quick brown fox anchors here." | Test DOCX parsing with formatting |
| `sample.md` | Markdown (GFM) | "The quick brown fox anchors here." | Test Markdown with tables, code, bold, links |
| `sample-bom.txt` | Text (UTF-8 BOM + CRLF) | "The quick brown fox anchors here." | Test BOM handling and line endings |
| `thai-ocr.png` | PNG (≥600px wide) | "การทดสอบระบบค้นหาข้อความภาษาไทย" | Test Thai text OCR |
| `latin-ocr.jpg` | JPEG (3+ text lines) | "The quick brown fox anchors here." | Test English text OCR |
| `corrupt.pdf` | PDF (truncated/invalid) | N/A | Test error handling for malformed PDFs |
| `oversized-marker.txt` | Text | N/A | Marker file for oversized content testing |

## Generation

All fixtures are generated by `generate.mjs`, which is idempotent and can be run multiple times safely.

```bash
node generate.mjs
```

### Dependencies
- `sharp` - Image generation (SVG → PNG/JPEG)
- `adm-zip` - DOCX creation (ZIP format)
- `mammoth` - DOCX text extraction (verification only)
- `pdf-parse` - PDF text extraction (verification only)
- `puppeteer` - PDF generation with proper text layer (Thai PDF)

All dependencies are available in the collector's `node_modules`.

### Browser Requirement
The Thai PDF (`digital-thai.pdf`) is generated using Puppeteer with Microsoft Edge. The script will:
1. Try to use Microsoft Edge at `C:\Program Files (x86)\Microsoft\Edge\Application\msedge.exe`
2. Fall back to hex-encoded PDF if Edge is not available
3. Report the method used in console output

## Verification

## Notes

### PDF XRef Warnings
Both `digital-latin.pdf` and `digital-thai.pdf` may produce "bad XRef entry" warnings from pdf-parse. This is expected due to hand-rolled xref tables. The PDFs are functionally valid and text extraction works correctly.

### Thai Text in PDF
`digital-thai.pdf` is generated using Puppeteer and Microsoft Edge to render proper Thai text with a real text layer. This ensures:
- Thai Unicode codepoints (U+0E00-U+0E7F) are present in the PDF text layer
- Text extraction tools can properly extract Thai text
- The PDF is compatible with text search and highlighting features

If Microsoft Edge is not available, the script falls back to a hex-encoded PDF (which may not have proper Thai text layer support).

### Image Quality
OCR images (`thai-ocr.png`, `latin-ocr.jpg`) are rendered at sufficient resolution (≥600px wide, ≥14pt font) for reliable OCR processing.
